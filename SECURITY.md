# Security

## Reporting

If you find a vulnerability or data exposure risk, please do not include sensitive material in a public issue.

Use GitHub's private vulnerability reporting if it is available for this repository. If it is not available, open a minimal public issue that says there is a security concern and avoids secrets, customer data, private product briefs, or unreleased assets.

## Data Safety

This repository should contain only public docs, fictional examples, validator code, and the packaged Skill.

Do not submit:

- private product briefs;
- unreleased product images or videos;
- customer or platform account data;
- credentials or environment files;
- real case-video assets unless you have permission to publish them.

The release validator scans for common local paths and credential-looking patterns, but contributors are still responsible for checking their own changes before opening a pull request.

## Supported Versions

Security fixes are applied to the current `main` branch.
