#### Prompt

Include a copied `【素材说明】` block in the prompt itself so the video model receives material roles. This block is required whenever the prompt uses `@图片` or `@视频`; explain every reference separately and briefly.

For standard A-class prompts, add the Unit Scope Contract directly after the material lines. For B-class prompts, the reference video controls the action timeline; do not rewrite that timeline in prose.

```text
【素材说明】
@图片1：P0，{是什么素材}，控制{产品身份/版本/空间/身体关系}，不控制{不应继承的内容}，对应{卖点/文案beat}
@图片2：P1，{是什么素材}，证明{卖点/部件/状态}，不控制{不应继承的内容}，对应{卖点/文案beat}
@图片3：P2，{是什么素材}，控制{场景/人物/道具/情绪}，不控制{产品状态/动作}，对应{文案实体}
@视频1：B，{是什么动作视频}，控制{动作/节奏/视角/状态变化}，不控制{背景/人物/其他}，对应{动作beat}

本段只做一件事：[标准 A 类 Unit 的唯一证明目标；B 类可改为参考视频优先说明]。
动作流：[标准 A 类用一段动作/运镜/情绪描述，不写具体秒段或逐秒分段；B 类不重写参考视频动作时间线]。
结束条件：[标准 A 类 clip 停在哪一帧；B 类按参考视频动作结果结束]。

【画面要求】
...
```

## Non-Negotiable Consistency

- Reference count in the material table equals prompt @ references.
- API reference counts equal material table counts.
- Preprocessing/upload list equals API counts.
- Production archetype is declared before core asset and Unit prompts.
- Golden case abstraction is declared before hallucination rules and Unit prompts.
- Core asset and TS risk table exists before Unit prompts.
- Script-Unit-material arrangement exists before Unit prompts.
- Script keyword/entity extraction exists before Unit prompts.
- A/B key rows are creator-confirmed or marked pending.
- Audio-visual sync audit exists after Unit prompts.
- Self-contained handoff check is present before final handoff.
- No unresolved placeholders remain.
- Every B unit names the reference video role and does not describe its action timeline.
- Every standard A-class unit declares unit goal, action flow, and end condition (U32); no segmented second-by-second prompt prose.
- Every unit declares product state and state consistency.
