# Why SKU Skills?

An ecommerce product video is not a generic creative prompt. It is a proof system.

A product-specific SKU Skill captures the rules that make a product video stable:

- what the product is;
- which states are valid;
- which parts move, open, fill, fold, heat, pour, stretch, or detach;
- which selling points need visual proof;
- which assets should control identity, action, scene, or rhythm;
- which model failures should never repeat.

## Why Not Just Prompt Templates?

Prompt templates collapse several separate decisions into one text block. That makes them quick to copy but hard to validate.

SKU Skills keep those decisions separate:

| Decision | Why it matters |
|---|---|
| Product structure | Prevents impossible states and wrong components |
| Selling-point proof | Keeps footage commercially meaningful |
| Asset role binding | Prevents reference images and videos from contaminating each other |
| A/B classification | Chooses whether a still, action video, or mixed reference strategy is safer |
| Bad-case regression | Turns failed generations into reusable guardrails |

## Who This Helps

- AI video producers making product videos.
- Ecommerce teams building repeatable short-video workflows.
- Agent Skill builders looking for a real multi-file Skill structure.
- Prompt engineers who need validation and production handoff, not only examples.

## The Practical Shift

The workflow moves from:

```text
copywriting -> prompt -> generation -> manual fix
```

to:

```text
brief -> product understanding -> selling-point proof -> material roles -> prompt-plan -> validation -> bad-case regression
```
