# Launch Playbook

Use this checklist when sharing the project publicly.

## Repository Setup

- Add a clear GitHub description.
- Add topics:
  - `agent-skills`
  - `codex`
  - `ecommerce`
  - `video-generation`
  - `prompt-engineering`
  - `aigc`
  - `sku`
  - `ai-video`
  - `workflow`
- Keep README first screen focused on the problem and the before/after.
- Keep examples fictional unless you have permission to publish product assets.

## Suggested Launch Copy

English:

```text
I open-sourced Ecommerce SKU Skill Builder: an Agent Skill system that turns product briefs, selling points, and reference assets into validated AI video-generation SKU Skill packages.

It is not a prompt-template dump. It adds product understanding, selling-point proof mapping, asset role binding, A/B state gates, prompt-plan validation, and bad-case regression.
```

Chinese:

```text
我开源了 Ecommerce SKU Skill Builder：一个把电商商品 Brief、卖点和素材编排转成可验证 AI 视频生成 SKU Skill 包的 Agent Skill 系统。

它不是普通 prompt 模板库，而是把商品理解、卖点证明、素材角色、A/B 状态、prompt-plan 校验和 bad-case 回归做成流程。
```

## Where To Share

- GitHub profile pinned repositories.
- X / Twitter.
- 即刻 / 小红书 / 知识星球.
- AI Agent and Codex communities.
- AI video workflow communities.
- Prompt engineering communities.

## What To Show

Show the transformation:

```text
Product brief + selling points + assets
-> product understanding
-> visual proof map
-> material roles
-> prompt-plan
-> validator
-> SKU Skill package
```

Do not lead with the internal file tree. Lead with the problem it solves.
